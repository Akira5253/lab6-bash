В репозитории два независимых Bash-скрипта, каждый из которых демонстрирует базовые конструкции языка (циклы, условия, функции), комбинирование стандартных Linux-утилит и базовую обработку ошибок.

| Скрипт                                               | Назначение                                        |
| ---------------------------------------------------- | ------------------------------------------------- |
| [`smart_backup.sh`](./smart_backup.sh)               | Умное резервное копирование директорий с ротацией |
| [`beer_to_war_thunder.sh`](./beer_to_war_thunder.sh) | Запуск War Thunder по звуку открытия банки        |

## Требования

- Linux / macOS / WSL2 (Windows Subsystem for Linux)
- Bash 4+
- Для `smart_backup.sh` - только утилиты из стандартного пакета (`tar`, `gzip`, `find`, `du`, `awk`, `sort`, `date`)
- Для `beer_to_war_thunder.sh` - дополнительно `ffmpeg`:
  ```bash
  sudo apt install ffmpeg     # Ubuntu / Debian / WSL
  brew install ffmpeg         # macOS
  ```

## Скрипт 1 - `smart_backup.sh`

Создаёт сжатые `tar.gz` архивы директорий с временной меткой в имени, проверяет целостность созданных архивов и автоматически удаляет старые копии (политика «оставить N последних»).

### Возможности

- Аргументы командной строки с валидацией через regex
- Журнал вывода с уровнями (`INFO`, `OK`, `WARN`, `ERROR`, `STEP`)
- Опциональное дублирование журнала в файл
- Поддержка исключений по шаблонам (`--exclude '*.log'`)
- Регулируемый уровень сжатия (1-9)
- Режим `--dry-run` - симуляция без реальных изменений
- Проверка целостности созданного архива через `tar --test`
- Автоматическое удаление повреждённых архивов
- Разные коды возврата для разных типов ошибок
- Строгий режим Bash (`set -euo pipefail`) и разные коды возврата для разных типов ошибок

### Примеры запуска

```bash
# Минимальный
./smart_backup.sh -s ~/documents -d /tmp/backup

# С исключениями и хранением 10 последних архивов
./smart_backup.sh -s ~/project -d /tmp/backup -k 10 \
    -e 'node_modules' -e '*.log' -e '.git'

# Симуляция + журнал в файл
./smart_backup.sh -s ~/important -d /tmp/backup -n -l /tmp/backup.log

# Полная справка
./smart_backup.sh --help
```

### Параметры

| Флаг                      | Описание                                           |
| ------------------------- | -------------------------------------------------- |
| `-s, --source <DIR>`      | Директория для резервной копии (обязательный)      |
| `-d, --dest <DIR>`        | Куда складывать архивы (обязательный)              |
| `-k, --keep <N>`          | Сколько последних архивов хранить (по умолчанию 5) |
| `-c, --compression <1-9>` | Уровень сжатия gzip (по умолчанию 6)               |
| `-e, --exclude <PATTERN>` | Исключить файлы/папки по шаблону (можно повторять) |
| `-l, --log <FILE>`        | Дублировать журнал в файл                          |
| `-n, --dry-run`           | Симуляция без реальных изменений                   |
| `--no-verify`             | Не проверять целостность архива                    |
| `-q, --quiet`             | Тихий режим (только ошибки)                        |
| `-h, --help`              | Справка                                            |
| `-v, --version`           | Версия                                             |

### Коды возврата

| Код | Значение                 |
| --- | ------------------------ |
| 0   | успех                    |
| 1   | общая ошибка             |
| 2   | неверные флаги           |
| 3   | нет исходной директории  |
| 4   | ошибка создания архива   |
| 5   | архив не прошёл проверку |

## Скрипт 2 - `beer_to_war_thunder.sh`

Слушает микрофон, ищет характерный звук открытия банки пива (короткий щелчок + протяжное шипение с преобладанием высоких частот) и при обнаружении запускает War Thunder через Steam.

### Как работает детектор

На каждое окно звука рассчитываются три независимых признака:

- peak - пиковая громкость (для щелчка язычка)
- rms - средняя громкость в окне
- hiss_ratio - разница RMS всего сигнала и RMS после фильтра высоких частот (>2 кГц). Чем меньше, тем сильнее преобладают высокие частоты - характерная подпись шипения

Срабатывание происходит, только когда все три условия выполнены одновременно. Это отсекает обычные хлопки, голос, музыку.

### Два режима работы

- Прослушивание микрофона (по умолчанию) — непрерывная запись и анализ звука в реальном времени.
- Анализ файла (`--file`) — разбор готовой аудиозаписи. Файл режется на окна той же длительности, каждое проходит через тот же классификатор. Удобно, когда живой микрофон недоступен, и даёт повторяемый тестовый кейс для демонстрации.

### Поддерживаемые платформы

| Платформа     | Захват звука    | Запуск игры                 |
| ------------- | --------------- | --------------------------- |
| Linux         | PulseAudio      | `steam` / `xdg-open`        |
| macOS         | AVFoundation    | `open steam://`             |
| Windows + WSL | ALSA (`hw:0,0`) | `cmd.exe /c start steam://` |

### Захват звука в WSL

ALSA пишет с устройства `hw:0,0`. USB-микрофон пробрасывается из Windows в WSL утилитой [usbipd-win](https://github.com/dorssel/usbipd-win):

```powershell
# В PowerShell (от администратора), один раз:
usbipd bind --busid <BUSID>
# После каждого запуска WSL:
usbipd attach --wsl --busid <BUSID>
```

Внутри WSL потребуется установить ALSA-утилиты и добавить пользователя в группу `audio`:

```bash
sudo apt install -y alsa-utils usbutils
sudo usermod -aG audio $USER   # требует перезапуска WSL
```

### Параметры

| Флаг                  | Описание                                                    |
| --------------------- | ----------------------------------------------------------- |
| `-f, --file <PATH>`   | Анализировать аудиофайл вместо микрофона                    |
| `-d, --device <NAME>` | Аудиоустройство (в WSL — ALSA-адрес, по умолчанию `hw:0,0`) |
| `-t, --chunk <SEC>`   | Длина окна анализа (по умолчанию 1.2)                       |
| `-p, --peak <DB>`     | Порог пика, dB (по умолчанию −12)                           |
| `-r, --rms <DB>`      | Порог RMS, dB (по умолчанию −28)                            |
| `-H, --hiss <DB>`     | Порог hiss-ratio, dB (по умолчанию 6)                       |
| `--cooldown <SEC>`    | Пауза после срабатывания (по умолчанию 30)                  |
| `--once`              | Выйти после первого срабатывания                            |
| `--calibrate`         | Записать 3 сек и подобрать пороги                           |
| `--test-launch`       | Только проверить запуск игры                                |
| `-n, --dry-run`       | Не запускать игру, только сообщить о детекции               |
| `-h, --help`          | Справка                                                     |
| `-v, --version`       | Версия                                                      |

### Примеры запуска

```bash
# 1. Откалибровать пороги под микрофон и банку
./beer_to_war_thunder.sh --calibrate

# 2. Режим с подобранными порогами
./beer_to_war_thunder.sh --peak -8.3 --rms -22.1 --hiss 4.5

# 3. Тренировка без запуска игры
./beer_to_war_thunder.sh --dry-run

# 4. Проверка, что Steam-протокол вообще работает
./beer_to_war_thunder.sh --test-launch

# 5. Запустится один раз и выйдет
./beer_to_war_thunder.sh --once

# 6. Справка по всем флагам
./beer_to_war_thunder.sh --help
```
